# trijbOS Agents

This directory defines the internal workstreams used to build `trijbOS`. Each file is a scoped brief for one agent role.

These agents are:

- `repo-audit.md`
- `product-architecture.md`
- `ui-ux-systems.md`
- `frontend-implementation.md`
- `persistence-data.md`
- `qa-performance.md`
- `documentation.md`

Use them as structured ownership guides:

- They do not replace source-of-truth docs in `docs/`.
- They do define what each workstream is allowed to own.
- They should be kept aligned with the MVP roadmap and architecture docs.
